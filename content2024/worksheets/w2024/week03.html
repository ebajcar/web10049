2.2. HTML document and website design and construction
  1. Document and website structure (MDN web docs)
    1. HTML for structuring content. HTML layout elements in more detail. (Information architecture: Information architecture, as applied to web design and development, is the practice of organizing the information, content, and functionality of a website so that it presents the best user experience it can, with information and services being easily usable and findable.)
    2. Basic sections of a document (header, navigation bar, main content, sidebar, footer)
    3. Many web designers consider the navigation bar to be part of the header rather than an individual component, but that's not a requirement; in fact, some also argue that having the two separate is better for accessibility, as screen readers can read the two features better if they are separate.
    4. we need to respect semantics and use the right element for the right job.
    5. article, section, aside, header, nav, footer, body, article, section; span, div; br, hr, p

2.5. Development process (Progressive Enhancement Phase I)

Needs update
Complete all steps.

(RECAP)
  1. Create project directory (folder) on local machine (client).
  2. Create content sub-directory.
  3. Create images sub-directory.
  4. Create css sub-directory.
  5. Create index.html in project directory.
  6. Start with minimal HTML5 document.
  7. Add viewport meta element.
  8. Add author and description meta elements.
  9. Add prologue (documentation header).
  10. Add your own comments. (Start with documentation: what needs to be done?)
  11. Validate at validator.w3.org and correct all warnings and errors.
  12. Add standard content page areas.
  13. Test using browser.
  14. Validate and correct all warnings and errors.
  15. Keep adding code fragments, testing, and validating until all content is marked up.
  16. Repeat steps 6 to 15 for each secondary file (stored in content sub-directory.
  17. Publish project using Filezilla on dev.fast.sheridanc.on.ca/syst10049.
  18. Validate for accessiblitity using wave.webaim.org. Correct all errors. Test with browser.
