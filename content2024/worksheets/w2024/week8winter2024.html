Moving to Phase 2 of Progressive Enhancement.
To focus on the new Learning Outcomes (red), we need to start with a well-formed, valid, semantically correct, and accessible HTML5 code (gray)

  7. Explain how web servers display web pages.
  8. Describe how to secure a web site.
  9. Use design techniques to lay out a user-friendly website.

Explore CSS modules (References)
  • CSS syntax (MDN) • The CSS syntax module describes, in general terms, the structure and syntax of cascading stylesheets, or CSS. It defines CSS as the language for describing the rendering of structured documents (such as HTML and XML), on the web and elsewhere. This module doesn't define any properties, data types, functions, or at-rules. Rather, it elaborates on how all of these features should be defined and how user agents should parse CSS. •
  • CSS cascade and inheritance (MDN) • Specificity • Cascade origin and importance • Values (actual, computed, initial, resolved, specified, used) • Origin types
  • CSS selectors (MDN) • The CSS selectors module defines the patterns to select elements to which a set of CSS rules are then applied along with their specificity. The CSS selectors module provides us with more than 60 selectors and five combinators. Other modules provide additional pseudo-class selectors and pseudo-elements. •
  • CSS backgrounds and borders(MDN) • Properties: background-attachment, background-clip, background-color, background-image, background-origin, background-position, background-repeat, background-size, background (shorthand) • border-color (shorthand), border-style (shorthand), border-width (shorthand), border (shorthand), border-collapse, border-radius (shorthand) , box-shadow
  • CSS colors (MDN) • color, opacity, background-color, accent-color • Values: rgb() - red-green-blue, rgba() - optional alpha-value [the alpha channel or transparency of a color], hsl() - hue-saturation-lightness, hsla() - optional alpha-value • Example: Color Syntax Converter

Activity 8.3. Practice. Practice. Practice.

Create a local project site called week8notes
Create a standard file organization in C:\public_html\syst10049. Use your home page as an overview and navigation to all the secondary pages. As you read and study the tutorials and modules in Activity 8.2. you come across examples. Implement each example as a secondary page. Use comments to cite where the example came from and annotate with your own observations, descriptions, and explanations. When you dig deeper and cross-reference to other sites, include notes on any connections and additional information you discovered. Use embedded styles (using STYLE element) for each page. Leave the home page unstyled. Validate HTML code, CSS code, accessibility. Pructice publishing.
Create a local project site called birdwatching
Create a standard file organization in C:\public_html\syst10049. Download the starting point file from Structuring a page of content. Place home page in the project root, images in the images folder, the CSS file in the css folder. Read through the Project Brief carefully and fully, then complete all the steps.
Once the home page is complete, copy it to the content folder to create the get-started.html, photos.html, blog.html, and gear.html. Change some of the main content sufficiently to identify which page the user is on. Remember, common page areas must remain the same for all pages (header, footer, and navigation). Update links in the navigation and test. Validate and publish. Test for accessibility and correct as required.
Create a local project site called biography
Create a standard file organization in C:\public_html\syst10049. Download the starting point file and organize all files appropriately. This project has only one HTML page, the home page. Read through the Project Brief carefully and fully, then complete all the steps. Once you have completed, tested, and validated the single-file solution, create a new file in the css folder and name it bio.css. Move all style rules to the bio.css file and replace the style element with an appropriate link element. Retest and revalidate. Publish. Test for accessibility and correct as required.
