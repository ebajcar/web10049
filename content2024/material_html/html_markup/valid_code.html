Well-formed and valid code

Well-formed code refers to compliance to standards, including

  1. closing all elements
  2. nesting elements properly
  3. using correct syntax for attributes

HTML5 only has a couple of exceptions to the above rules:

  1. Empty elements: Empty elements are not closed. Examples: BR, HR, META, IMG, etc.…
  2. Attribute minimization: single-valued attributes are minimized. Examples: autofocus, controls, etc.…

In addition to syntactical errors, we need to consider semantic correctness and accessibility principles.

Validity refers to eliminating all syntax and semantic errors. This can be done by using the validation tools available from W3C.
