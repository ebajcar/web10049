My Hypertext Markup Language (HTML) Checklist

HTML5 SYNTAX
Basic
<tagname attribute="value" attribute="value">CONTENT</tagname>
WELL-FORMED AND VALID HTML5
Basic Rules
  1. All elements are properly closed. <p> content </p>
  2. Empty elements are never closed. [exception] <br>
  3. All elements are properly nested. <p><em><mark> content </mark></em></p>
  4. All attributes have a quoted value assigned. <img src="image.png" alt="an image">
  5. Single-valued attributes are minimized. [exception] <input autofocus selected>
Semantic correctness all elements are chosen well.

PURPOSE
HTML or Hyper Text Markup Languages

First stage of PE (or input/output design) to 1. identify the characteristics and nature of content, based on which observations, 2. select the most appropriate container(s) to mark up the content. The objective is for the markup to best describe the characteristics of the content.

Semantics are front and center with HTML5, giving meaning to structure of web apps and dynamic web content.
