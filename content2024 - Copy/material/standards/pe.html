Progressive Enhancement (PE)

PE is a strategy for web design that emphasizes core web page content first. It emphasizes accessiblity, semantic HTML markup, and external stylesheet and scripting technologies. PE supports the development process in many ways, including:

  1. Preparation and organization of files.
  2. Creating HTML documents.
  3. Identifying content types (characteristics) and marking up appropriately (semantic correctness).
  4. Testing and observing (default) presentation.
  5. Validating, correcting to remove errors, and re-testing.

PE consists of core principles:

  1. basic content should be accessible to all web browsers
  2. basic functionality should be accessible to all web browsers
  3. sparse, semantic markup contains all content.
  4. enhanced layout is provided by externally linked CSS
  5. enhanced behaviour is provided by unobtrusive, externally linked JavaScript.
  6. end-user web browser preferences are respected

Read further Progressive enhancement (wikiwand)

The PE process consists of three steps:

  • Step One: Build solid foundation with HTML5. Identify content and use the most suitable (meaningful) markup to organize it. All content must be accessible without styles.
  • Step Two: Using well-formed and valid HTML5, style the elements for best presentation. All content must be accessible without scripts.
  • Step Three: Using well-formed and valid HTML5 styled with well-formed and valid CSS3, add behaviour with Javascript.
