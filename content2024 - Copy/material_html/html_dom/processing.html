DOM processing

Every time the browser processes HTML markup, it goes through the process. This process can take some time, especially if we have a large amount of HTML to process.

STEP 1: Conversion — convert bytes to characters
The browser reads the raw bytes of HTML off the disk or network, and translates them to individual characters based on specified encoding of the file (for example, UTF-8).
STEP 2: Tokenizing — identify tokens
The browser converts strings of characters into distinct tokens—as specified by the W3C HTML5 standard for example, "<html>", "<body>" —and other strings within angle brackets. Each token has a special meaning and its own set of rules.
STEP 3: Lexing — convert tokens to nodes
The emitted tokens are converted into "objects," which define their properties and rules.
STEP 4: DOM construction — build the DOM tree
Finally, because the HTML markup defines relationships among different tags (some tags are contained within other tags) the created objects are linked in a tree data structure that also captures the parent-child relationships defined in the original markup: the HTML object is a parent of the body object, the body is a parent of the paragraph object, and so on.
