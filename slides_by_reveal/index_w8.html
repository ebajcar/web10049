Server-side Programming

PHP Hypertext Preprocessor (PHP) is an open source server-side scripting language designed for web develoment.

PHP Language

  1. a PHP script executed on the server
  2. plain HTML result is sent back to browser.
  3. a PHP script starts with <?php and ends with ?>
  4. File extension for PHP files is .php
  5. Primary source: PHP Manual at php.net

PHP programs

Basic rules

When PHP interpreter parses a file,
it looks for opening and closing tags 

<?php
// declare and initialize variable txt to store a value 
$txt = "Hello world!";
 
// display the stored value in variable txt
echo $txt;
?>

Hybrid PHP

Everything outside of a pair of opening and closing tags is ignored by the PHP parser which allows PHP files to have mixed content (hybrid). 

<html>
<body> 
<?php
$txt = "Hello world!";
?>
<h1>
<?php echo $txt; ?>
</h1>
</body>
</html>

Other open tags

<? PHP also allows for short open tag <? which is discouraged since it is only available if enabled and conflicts with other XML tags.
<?= 
<?= $someVar ?>
// is available shortcut for
<?php echo $someVar ?>

File extension

  • On most servers, the default extension for PHP files is .php.
  • If the server supports PHP, then you do not need to do anything.
  • Files created with the .php extension located in the web directory (webspace) will be automatically parsed by the server: http://userid.dev.fast.sheridanc.on.ca/path/file.php
    where the web directory is /home/userid/public_html/path

PE Principles

What distinguishes PHP from something like client-side JavaScript is that the code is executed on the server, generating HTML which is then sent to the client. The client would receive the results of running that script, but would not know what the underlying code was.

Always validate the generated HTML!
